Pocket D&D — multi-tool event loop для инвентаря

Что исправлено:
1. За один ход модель может последовательно вызвать до 4 inventory tools.
2. Каждый tool call синхронно проходит через существующий Java NativeToolCallback.
3. Java валидирует изменение, меняет Room и создаёт отдельную StateChange-плашку.
4. Tool response декодируется в том же llama_context, после чего модель может вызвать следующий tool.
5. Raw <|tool_call> больше не должен попадать в пользовательский текст.
6. Неизвестный/незавершённый поздний tool подавляется и логируется.
7. Добавлен динамический контроль остатка контекста, чтобы дополнительные tool responses не переполнили n_ctx.

Ограничение:
MAX_INVENTORY_TOOL_CALLS_PER_TURN = 4.

Минимальный тест:
PLAYER: Я кладу *карту* и *бусину* в инвентарь

Ожидается:
- два TOOL APPLIED в MyDND_ONEPASS;
- native EVENT TOOL_CALL #2 для второго изменения;
- две отдельные плашки;
- оба предмета в инвентаре;
- ни одного видимого <|tool_call> в чате.

Дополнительный тест:
PLAYER: Я выбрасываю *карту* и беру *монету*

Ожидается две независимые плашки REMOVE и ADD.
