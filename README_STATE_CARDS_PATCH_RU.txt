POCKET D&D — ПЛАШКИ СОСТОЯНИЯ / UNDO / NPC MEMORY / CHECKS / HELP

Что добавлено:
1. Универсальная таблица state_changes (Room version 7).
2. Плашки: предмет получен/потерян, NPC запомнил, нужна проверка, событие мира.
3. Undo для безопасных изменений: inventory ADD/REMOVE и NPC memory.
4. Отменённая плашка становится серой и сохраняется в истории.
5. Старый Undo блокируется, если тот же предмет/NPC уже изменился позже.
6. Отношение NPC -10...+10 остаётся скрытым в БД; игрок видит только факт памяти и общий тон.
7. CHECK создаёт кнопку «БРОСИТЬ d20»; результат остаётся на плашке и попадает в prompt следующего хода.
8. Главное меню → «?» открывает справку и сведения об авторе.
9. README.md обновлён.

Применение:
Распаковать ZIP в корень MyDND с заменой файлов.

Проверка:
bash verify_state_cards_patch.sh /home/enhort/AndroidStudioProjects/MyDND

Сборка:
./gradlew assembleDebug

Минимальный тест:
A. Я беру *железный ключ* → плашка → ↶ ОТМЕНИТЬ → ключ исчез, плашка серая.
B. Значимый поступок перед NPC → при NPC_MEMORY появляется «NPC ЗАПОМНИЛ», числовое отношение не видно.
C. Рискованное действие → при CHECK появляется «БРОСИТЬ d20» → результат остаётся на плашке.
D. Главное меню → ? → проверить справку и автора.
