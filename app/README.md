# MyDND

**MyDND** — Android-приложение на Java/XML для локального офлайн-мастера настольной RPG в духе DnD.

Проект разрабатывается как мобильный помощник для ведения кампании: приложение хранит состояние мира, персонажа, события, NPC, квесты и инвентарь локально, а локальная LLM используется для генерации атмосферных ответов мастера.

## Идея проекта

Главный принцип архитектуры:

* LLM отвечает за повествование, атмосферу, диалоги NPC и описание сцен.
* Java-код отвечает за правила, броски кубиков, состояние персонажа, инвентарь, квесты, NPC, журнал событий и память кампании.
* Долгосрочная память кампании не хранится в контексте модели, а сохраняется локально в SQLite/Room.
* В промпт передаются только релевантные данные: текущая сцена, состояние персонажа, активные NPC, активные квесты, последние события и важные факты.

## План MVP

Минимальная версия приложения должна включать:

1. Создание кампании.
2. Создание персонажа.
3. Экран чата с мастером.
4. Ввод действия игрока.
5. Ответ мастера от LLM или временной заглушки.
6. Броски кубиков внутри приложения.
7. Журнал событий.
8. Хранение NPC, локаций, предметов и квестов.
9. Краткие summary сцен.
10. Подмешивание в промпт только релевантных фактов, а не всей истории.

## Архитектурные модули

Планируемая структура проекта:

```text
ui/
llm/
game/
rules/
memory/
prompt/
db/
util/
```

Основные модули:

* `LlmEngine` — загрузка модели, генерация ответа, streaming, отмена генерации.
* `PromptBuilder` — сбор промпта из состояния кампании.
* `GameState` — текущее состояние кампании.
* `CampaignMemory` — журнал, summary, факты, история решений.
* `DiceService` — броски кубиков и проверки.
* `RulesEngine` — базовые правила DnD.
* `EventExtractor` — выделение изменений из ответа модели.
* `Database/DAO` — работа с SQLite/Room.
* `UI` — экраны кампаний, чата, персонажа, журнала, NPC, квестов и настроек.

## Игровой цикл

```text
игрок вводит действие
        ↓
приложение собирает состояние кампании
        ↓
PromptBuilder формирует промпт
        ↓
LlmEngine генерирует ответ мастера
        ↓
приложение сохраняет событие
        ↓
при необходимости обновляет состояние мира
```

## DiceService

Нейросеть не должна сама бросать кубики.

Если нужна проверка, модель должна запросить её, а приложение выполняет бросок и возвращает результат в следующий ход.

Планируемая поддержка:

* d4
* d6
* d8
* d10
* d12
* d20
* d100
* модификаторы
* проверка против сложности

Пример:

```text
Проверка: Ловкость
Сложность: 14
Бросок: d20 + модификатор
```

## Стиль мастера

Базовый стиль ответов:

* русский язык;
* мрачное фэнтези;
* атмосферно, но без лишнего пафоса;
* краткие ответы;
* не решать за персонажа игрока;
* не ломать факты кампании;
* помнить последствия действий;
* если действие рискованное — просить проверку;
* явно указывать важные изменения: предмет получен/потерян, изменилось HP, появился NPC, начался квест, изменилась локация.

## Базовый системный промпт

```text
Ты — мастер настольной RPG в духе DnD. Веди сцену, описывай мир, играй NPC и реагируй на действия игрока. Не решай за персонажа игрока. Если нужна проверка, запроси её. Не бросай кубики сам. Учитывай текущую сцену, NPC, квесты, инвентарь, журнал событий и состояние персонажа. Отвечай на русском языке. Пиши атмосферно, но кратко. Если появляются важные изменения, укажи их явно.
```

## Пример промпта на ход

```text
SYSTEM:
Ты — мастер настольной RPG. Не решай за игрока. Если нужна проверка, запроси её.

CAMPAIGN:
Название: {campaignTitle}
Жанр: мрачное фэнтези

CURRENT_STATE:
Локация: {location}
Сцена: {scene}
Персонаж: {characterState}
Активные NPC: {activeNpcs}
Квесты: {activeQuests}
Инвентарь: {inventory}

MEMORY:
Последние события:
{recentEvents}

Релевантные факты:
{relevantFacts}

PLAYER_ACTION:
{userInput}

TASK:
Ответь как мастер. Если нужна проверка, явно напиши тип проверки и сложность.
```
